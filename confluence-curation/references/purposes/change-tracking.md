# 변경 추적 + 트렌드 감지

## 메타

- purpose: `change-tracking`
- 한글 라벨: 변경 추적
- 트리거 문구: "최근 변경", "뭐가 바뀌었", "변경 이력", "트렌드", "활동 증가", "새로 생긴", "업데이트가 많", "what changed", "recently"

## 핵심 질문

- 최근에 어떤 문서가 변경되었는가?
- 특정 기술이나 주제의 활동량이 증가하고 있는가?
- 회사 내에서 관련 프로젝트가 새로 생기고 있는가?
- 누가 주도적으로 변경하고 있는가?

## 출력 섹션 (순서대로)

1. **요약** — 전체 변경 규모, 주요 트렌드 1-2문장
2. **트렌드 신호** — 활동량 급증 문서/주제, 신규 생성 문서, 기여자 증가 패턴
3. **변경 타임라인** — 시간순 주요 변경 (최대 30건), 각 변경에 주체 명시
4. **변경 주체 분석** — 누가 어떤 문서를 변경했는지, 새 기여자 등장 여부
5. **문서 현황 표** — 상태 컬럼 대신 "최근 변경 빈도" 컬럼 포함
6. **후속 확인 필요 항목** — 추가 모니터링이나 팔로업이 필요한 사항

## 인사이트 강조 규칙

- `derive_conclusion()`: "어떤 문서가 기준인가" 대신 "어떤 변화가 일어나고 있는가" 프레이밍
- `derive_actions()`: "아카이브/통합" 대신 "팔로업/모니터링/관련 프로젝트 확인" 액션 우선
- `recent_changes`와 `maintainer_signals`를 최대한 확장 표시
- 충돌 분석, 신뢰도 세부 분석은 축소

## 리뷰 가중치

- freshness reviewer: 2배 가중 (최신성이 핵심)
- trust reviewer: 표준
- contradiction reviewer: 표준
- executive reviewer: 표준

## 축소 대상

- 신뢰도 세부 분석 (trust score 비교 등)
- 중복 정리 액션 (아카이브/리다이렉트 권고)
- 소스 오브 트루스 판정
