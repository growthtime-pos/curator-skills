# 공통 출력 규칙

## 기본 원칙

- 최종 출력은 항상 한글로 작성한다
- 점수보다 근거 설명이 더 중요하다
- 최신성과 신뢰도는 분리해서 보여준다
- 애매한 경우 억지로 하나의 정답 문서를 고르지 않는다
- 모든 주요 인사이트에는 근거 페이지와 짧은 증거가 함께 제시되어야 한다

## 상태 라벨

- `최신·신뢰 높음`
- `최신이나 검증 부족`
- `신뢰 높으나 오래됨`
- `중복 가능성`
- `대체됨 가능성`
- `검토 필요`

## 권장 어투

- `~로 보입니다`
- `~가능성이 높습니다`
- `현재 기준으로는`
- `근거가 충분하지 않습니다`
- `추가 확인이 필요합니다`
- `실무 기준으로는`
- `배경 문맥으로는`
- `상충하는 근거가 있습니다`

## 확신도 표현

- `높음`: 최신성·신뢰도 모두 강하고 충돌이 낮을 때
- `보통`: 유력한 답은 있으나 일부 신호 불완전
- `낮음`: 미해결 충돌, 근거 약함, 휴리스틱 의존

## 마크다운 규칙

- 섹션은 `##` 레벨 사용
- 하위 주제는 `###` 레벨 사용
- 테이블은 파이프 구분자 사용
- 근거 인용은 백틱으로 문서명 감싸기
