# 온보딩 / 학습

## 메타

- purpose: `onboarding`
- 한글 라벨: 온보딩
- 트리거 문구: "처음", "온보딩", "학습", "정리해줘", "어디서부터", "overview", "getting started", "입문", "개요"

## 핵심 질문

- 이 주제를 처음 접하는 사람이 어디서부터 읽어야 하는가?
- 핵심 내용을 빠르게 파악하려면 어떤 포인트를 봐야 하는가?
- 이 주제의 역사적 배경과 맥락은 무엇인가?
- 관련된 다른 문서들은 어떤 구조로 연결되어 있는가?

## 출력 섹션 (순서대로)

1. **주제 요약** — 비전문가 관점에서 이 주제가 무엇인지 1-2단락 요약
2. **추천 읽기 순서** — 번호 매긴 문서 리스트, 각 문서를 왜 이 순서로 읽는지 이유 설명
3. **핵심 내용 정리** — 본문에서 추출한 핵심 포인트 (bullet)
4. **배경 문맥** — 역사적 맥락, 관련 의사결정 배경, 오래된 문서의 가치 설명
5. **문서 맵** — 문서 간 관계를 계층/그룹으로 시각화
6. **추가 탐색 제안** — 관련 space, label, 키워드 추천

## 인사이트 강조 규칙

- `derive_conclusion()`: "기준 문서" 대신 "시작점 문서" 프레이밍
- `derive_actions()`: "정리" 대신 "이 문서를 먼저 읽으세요", "다음으로 이것을 읽으세요", "담당자에게 문의" 액션 우선
- 본문 요약(evidence_snippets)을 최대한 확장
- 충돌 정보는 최소화하고 학습 경로 중심으로 재구성

## 리뷰 가중치

- freshness reviewer: 표준
- trust reviewer: 표준
- contradiction reviewer: 표준
- executive reviewer: 2배 가중 (읽기 쉬운지, 실행 가능한지)

## 축소 대상

- 충돌 분석 (conflict_notes)
- 스코어링 세부 (freshness/trust score 비교)
- 중복 정리 액션 (아카이브/리다이렉트 권고)
- 변경 타임라인 세부
