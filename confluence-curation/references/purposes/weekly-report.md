# 주간보고

## 메타

- purpose: `weekly-report`
- 한글 라벨: 주간보고
- 트리거 문구: "주간보고", "이번 주", "weekly report", "특정 인물", "누가 올린"

## 핵심 질문

- 지정한 기간에 어떤 문서가 새로 작성되거나 수정되었는가?
- 지정한 인물들이 어떤 페이지에 관여했는가?
- 이번 주 보고에 넣을 핵심 변경과 다음 주 확인점은 무엇인가?

## 권장 수집 조건

- `--page-url` 또는 `--root-page-id` 로 기준 페이지를 지정한다.
- 기준 페이지 자체도 포함해야 하면 `--include-root-page` 를 사용한다.
- 정확한 보고 기간은 `--updated-from YYYY-MM-DD` 와 `--updated-to YYYY-MM-DD` 로 지정한다.
- 대상 인물은 `--contributors` 에 쉼표 구분으로 입력한다. account id, 표시 이름, public name, email 과 매칭한다.
- 내용 요약이 필요하면 `--include-body` 를 사용한다.

## 출력 섹션

1. **보고 범위** — 기준 페이지, 기간, 대상 인물, 수집 문서 수
2. **이번 주 핵심 요약** — 보고에 바로 넣을 수 있는 변경 요약
3. **주제별 변경 동향** — 관련 문서 묶음별 결론과 후속 조치
4. **인물별 활동** — 대상 인물별 관여 문서와 링크
5. **주요 변경 타임라인** — 기간 내 주요 업데이트 흐름
6. **문서별 상세** — 문서명, 최근 수정일, 매칭 인물, 판단 근거
7. **다음 주 확인할 점** — 불확실성, 누락 가능성, 확인 필요 액션

## 해석 규칙

- "올린 페이지"는 생성자뿐 아니라 최근 수정자와 버전 이력의 수정자까지 포함해 해석한다.
- 지정 인물과 매칭되지 않는 문서는 주간보고 대상에서 제외한다.
- 날짜 범위가 주어지면 updated-at이 없는 문서는 제외한다.
- 이름 매칭은 Confluence 프로필 수집 결과에 의존하므로, 프로필 API가 제한되면 account id 또는 email 지정이 더 안정적이다.
